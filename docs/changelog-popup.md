# Changelog popup

How the "Changelog" window that appears on the login screen after a client update is produced.

## Source

- `changelog.txt` at repo root is the authoring file.
- `build.xml:185` bundles it into `hafen.jar` (so it's reachable via `getResourceAsStream("/changelog.txt")`).

## Trigger

`src/haven/Config.java:72-78` (static init):

```java
static {
    loadBuildVersion();
    isUpdate = !CFG.VERSION.get().equals(version) || !getFile("changelog.txt").exists();
    if(isUpdate){
        CFG.VERSION.set(version);
    }
}
```

- `version` is read from the `/buildinfo` resource, generated by the `buildinfo` target in `build.xml:30-40` as `"<build-time> <git-rev-short>"`. Every rebuild produces a new string.
- `CFG.VERSION` (`src/haven/CFG.java:18`) is the last-seen version, persisted in the user's config JSON.
- `isUpdate` is true when either:
  - persisted version differs from the current build's version, or
  - the user has no local `<homedir>/changelog.txt` on disk yet.

Setting `CFG.VERSION` to the new version in the same static block ensures the popup fires once per build.

## Display

`src/haven/LoginScreen.java`:

- `added()` (450-456) calls `showChangeLog()` when `Config.isUpdate` is true, so the popup appears on the login screen.
- `showChangeLog()` (79-104):
  - Creates a `WindowX` titled `"Changelog"` at screen (50,50) containing a 450x500 `Textlog`.
  - Reads `/changelog.txt` from the jar, appends up to 200 lines to the textlog, and simultaneously writes those lines to `<homedir>/changelog.txt`. That on-disk copy is what the `!getFile(...).exists()` check in `Config` later looks for.
  - Calls `setprog(0)` so the textlog is scrolled to the top.
- `destroy()` (468-474) tears the window down with the login screen.

## Gotchas

- Every `ant bin` rebuild bumps the version string (build-time + git short hash), so any rebuild re-triggers the popup on next launch regardless of whether `changelog.txt` actually changed.
- There is no in-game button to re-open the window; the only trigger is the auto-show on login.
- The i10n string `"Show Client Changelog"` in `i10n/ko/button.json` is a dead Kami-era leftover - no Java code references it.
